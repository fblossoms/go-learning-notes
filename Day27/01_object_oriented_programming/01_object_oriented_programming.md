# 面向对象
- 认识世界的方法论
  - 把万事万物抽象成类（分门别类）
    - 例如
      - 人类，抽象的概念，是人类特征的集合，不代表具体的人
      - 人，往往意思是某个人，某个具体的人，某个具体实实在在的人，他是人类这个抽象概念的具体化的其中一个
    - 从抽象概念c -- 实例化 -> 具体的实例instance、object，实例化
    - 实例和实例发生相互作用

  - 一切皆对象
  - 区别于面向过程，面向对象用于理解系统，理解相互作用的关系
- Go不是严格意义上的面向对象语言，用最低的成本实现了类似的功能。很多人认为这就是Go的面向对象

- 人吃鱼场景
  - 面向对象
    - 具体的那个人 吃 具体的那条鱼
    - 吃是动作，也是操作、方法
    - 过程
      - 抽象出
        - 人类

```txt
                    class Person
```

          - 吃（动作，属于人类的共同特征）
            - 把吃的方法函数体步骤列出来

```txt
                            eat(anything any)
                                switch any.(type) // 吃什么，肉？饭？
                                    case 某类型1:  // 吃的过程不一样，引入面向过程
                                    case 某类型2:  // 吃的过程不一样，引入面向过程
```

        - 鱼类

```txt
                    class Fish
```

          - 品种（属性，是数据特征）
        - （在实际构建类型时，只会把系统关系的特征实现）
      - 实例化
        - 需要构造器构造实例（没有实现构造函数，构造函数，会有默认的）
          - 构造函数（Go中的构造函数不是类的方法，是普通函数）
            - 习惯上，构造函数命名要以New或new开头。如果有多个构造函数，可以使用不同命名函数，因为Go也没有函数重载
            - 构造函数为了得到快速得到实例或实例的指针
        - 实例消亡时，会调用析构函数（解析、分解）
        - 示例语法
          - 具体的那个人  var p = Person()
          - 具体的那条鱼  var f = Fish()
          - 具体的那个人发出了吃的动作作用在具体的那个鱼上 p.eat(f)
            - 引入面向过程

  - 面向过程
    - 吃的过程是什么
  - 面向对象和面向过程不是对立的关系，只是分析的角度不一样

- 三要素
  - Go形似面向对象，有面向对象的功能，就可以应用面向对象的设计思想的开放项目了
  - 封装（把属性和方法规整到类）
    - 类
      - 属性，字段，数据成员，有类型，int、string、[]int等
      - 方法，操作、动作，成员
    - 访问控制
      - 通过结构体，可以把数据字段封装在内，还可以为结构体实现方法
        - Go通过首字母大小写，在一定程度上实现了public、private的访问控制
          - 1、属性、方法标识符首字母大写，实现了对包外可见的访问控制
          - 2、属性、方法标识符首字母小写，仅包内可见
      - 属性、方法是否暴露？暴露程度
      - 隐藏该隐藏的，暴露该暴露的
        - public
        - private
        - protected
  - 继承
    - 父类有，子类就可以直接继承得到
      - 避免重复代码
    - 子类的实例属于自己类，也属于父类，但是不能说反着说
    - 覆盖override
      - 子类觉得继承的不好，决定自己实现
        - 1、完全覆盖，不想和父类一样
        - 2、部分覆盖（锦上添花），借用部分父类，做改进
  - 多态
    - 同一个方法在不同实例上表现不同
    - 前提：有继承、覆盖
    - 父子继承、覆盖
      - 父类变量被赋值为子类实例，用这个父类变量调用覆盖的方法，调用子类的方法
      - 不同子类实例赋值给了父类变量，父类变量调用被覆盖的方法时，会根据实际的子类类型调用方法
