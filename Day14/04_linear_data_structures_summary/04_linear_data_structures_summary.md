# 常见线性数据结构

- 分类
  - 顺序表：array
  - 链接表：linked list，double（双向链表）
  - 栈（逻辑结构）：stack堆栈（其实是栈）
    - 后进先出LIFO（烤串）
      - 大多情况下使用链接表，少数采用顺序表
      - 若用顺序表，会引起频繁扩容，也会引起大规模数据浪费，由于长度固定，使用完后，不会自动回收
  - 队列（逻辑结构）：queue
    - 先进先出FIFO（排队）
      - 不允许中间操作，操作在两端，元素一旦删除就不在队列中了。如双端队列deque，一定是链接表实现
      - 优先队列：小顶堆，不是线性结构
      - 环形队列：线性结构

    - 可以在本进程、本机、消息队列中间件（服务，单节点，多节点分布式 *mq、kafka）使用
    - 作用:
      - 解耦：异步
      - 拦洪大坝（慢慢向下游输出）即控制消费速度，平滑生产速度

    - 生产者、消费者模型
      - 伪代码

```go
for // 循环：同步
    a = produce()   // 生产值给a（生产者）
    a各种操作   // 使用a（消费者）
生产者被迫等待消费者，造成耦合过深。需要解耦：
var q queue
// 使用并发 concurrency
线程 1 跑一个循环
    for
        q.push(produce())   // 不需要等待消费者消费，只管生产
线程 2 跑一个循环
    for
        a = q.get() // q没有数据就会阻塞，有数据就get
线程 3 跑一个循环
    for
        a = q.get() // q没有数据就会阻塞，有数据就get
```
