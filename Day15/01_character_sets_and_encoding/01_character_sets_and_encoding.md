# 字符串和编码

计算机世界中都是0和1组成的数字，看到一个字节的数字或者多个字节的数字，赋予它类型，才能让计算机知道该怎么读

人为建立编码表，把数字（0和正整数） -> 符号（人类能看懂的符号，后续还需要找到字体文件，符号对应点阵，送到显卡，绘制到输出的显示器。
找不到对应的文件就会显示其他字体，也就是乱码）

形成的表称为字符集charset

形成一种标准，有名的有

- ASCII（美国信息交换标准代码）
  - 英文符号、标点、控制字符
  - 总共128个符号，1个字节，低7位，0~127。还剩最高位是1的128种状态，兼容法语、西班牙语、俄语等
    - 0x9     int整数类型
    - '0x9'   字符rune类型，本质是int32
    - "0x9"   字符串string类型
  - 重

| 序号 | 转义字符 | 十进制数 | 说明 |
| --- | --- | --- | --- |
| 1 | \x00 | 0 | null字符，表中第一项，C语言中的字符串结束符 |
| 2 | \x09 \t | 9 | tab字符 |
| 3 | \x0d \x0a \r\n | 13 10 | 回车和换行 |
| 4 | \x20 | 32 | 空格 |
| 5 | \x30~\x39 | 48~57 | 字符0~9 |
| 6 | \x31 | 49 | 字符1 |
| 7 | \x41 | 65 | 字符A |
| 8 | \x61 | 97 | 字符a |

- UTF-8、GBK都兼容了ASCII

# 字符比较大小

Go中字符本质是int32，所以实际上是整数（ASCII）比较大小

- 例如'A' > 'a' // false，因为ASCII中'a'对应97（十进制），'A'对应65（十进制）

# 字符串比较大小

从左到右，对应位置索引的字符比较，如果当前位置已经决出胜负，就不用往后比了

- 例如"abccc" > "aaccc" // true，索引0中，'a'对应的数值相同（97），到索引1中，'b'（98） > 'a'（97）

# 单字节编码

一篇文章就一段连续的整数，要把它们每个字节都去查编码表，一次只能对应一种编码，每个字节的这个整数对应的符号就知道

# 中文编码问题

1个字节不够，需要2个字节。用到DBCS 双字节字符集

# DBCS

- GB2312
  - 2Byte 整数 -> 中文字符。兼容ASCII，每个字节最高位都是1
  - 收录文字不多，跟不上时代
- GBK（GBK是GB2312的超集）
  - 低字节 > 127 0x80 ~ 0xff，高字节不做要求00 ~ ff
  - 收录文字还是较少，但基本够用。是Windows的默认字符集
- GB18030（GBK的超集）

# 码点冲突：同一个码点在不同字符集当中对应不同的符号，同一片文章种乱码

所以要建立一个统一的不冲突的编码表，同时收录全世界的已知标准符号，预留一些空位应对未来发展

- Unicode字符集
  - 正整数 -> 符号
  - 兼容ASCII
  - 0x00 ~ 0x10ffff，至少3个字节。为了对齐可以考虑四个字节

  - 编码
    - 序列化、反序列化，决定如何存储、传输，字节序
    - UTF（Unicode Transformation Format）
      - UTF16：边长编码格式，码点 > \xFFFF（2Byte）则采用4Byte。否则采用2Byte
      - UTF32：4Byte
      - UTF8：变长编码格式
        - <= \x007F 使用 1 个字节
        - \x007F 且 <= \x07FF 使用 2 个字节
        - \x07FF 且 <= \xFFFF 使用 3 个字节
          - 汉字块在 \x4E00 ~ \x9FFC 之间，因此使用 3 个字节
        - \xFFFF 使用 4 个字节

# 大小端 big little endian

例如：27979（十进制） 0x6d4b（十六进制），4b是整数的尾部

```txt
内存地址（假设）    1000    1001 --> 增大
                 6d      4b       大尾，4b放在大地址。字符串是大尾，个字面量书写一致，网络传输
                 4b      6b       小尾，4b放在小地址。计算机内存多采用
```

# 文件的本质

躺在磁盘上的字节序列

- 如果该字节序列能够使用编码解读成功，显示成为人类可理解的符号，那就是文本文件
- 如果该字节序列不能使用编码解读，称为二进制文件
- 某种扩展名文件，可以理解为某种格式的文件
