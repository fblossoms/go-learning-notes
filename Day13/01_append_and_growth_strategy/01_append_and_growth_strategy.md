# append  把切片看作容器，append指的是在容器内部的尾部追加（after：容器（外部）的后面）

- append是切片slice的专用方法
- 作用是为一个指定的切片的尾部追加新元素，返回一个新切片（header）
- len增加，可能会超过cap，不能扩容当前数组（数组初始化后长度固定不可变），所以要另辟蹊径在其他地方放底层数组

# 扩容策略（扩充的是容量cap）

- GOROOT/src/runtime/slice.go growup
- 新长度len = 当前切片的len + append的元素个数
- 旧容量cap = 当前的容量cap
- 扩容条件：底层数组容量不够，即当前切片的len + append的元素个数 > 当前的容量cap。会触发扩容机制
- 扩容操作：将生成新的能够存放新长度的底层数组，有新的、足够的容量cap，并拷贝原来的切片中元数据指定元素们到新数组来，然后尾部追加新的元素
- 策略
  - if 若新长度len > 2倍的旧容量cap，则新容量cap == 新长度len（你说多长就多长，总是一个向上的整偶数）
  - else
    - if 若旧容量cap < 阈值（256），则新容量cap == 两倍的旧容量cap
    - else 若旧容量cap >=  阈值（256），则新容量cap 约等于 1.25倍的旧容量cap（1.18版本后）
