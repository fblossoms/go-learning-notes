# 程序控制

- 顺序：按照先后顺序一条条执行

- 分支：根据不同情况判断，条件满足执行某条件下的语句
    - if-else写法：表达式写法的作用是限定变量/常量在当前if-else中使用
        - 单分支

```go
if 布尔值 {    // 左括号必须写在当前if行
    // n条语句，可以是0条语句，布尔值为true时才进入语句
}
```

============================================================

```go
if 表达式; 布尔值 {    // 表达式后必须写分号（;），左括号必须写在当前if行
    // n条语句，可以是0条语句，布尔值为true时才进入语句
}
```

============================================================

    - 多分支
      - 双分支

```go
if 布尔值 {
    // true时语句
} else {
    // false时语句
}
```

- 多分支：由上向下逐个测试if或else if，如果当前条件成立（true），则进入该条件语句，下面的条件将不会再测试（多分支最多执行一个条件），都不满足则进入else语句。前一个分支条件被测试过，下一个条件相当于隐含着这个被测试的条件，所以要注意是否有逻辑漏洞

```go
if 布尔值 {
    // true时语句
} else if 布尔值 {
    // true时语句
} else if 布尔值 {
    // true时语句
} else {
    // false时语句
}
```

- 可以不写else兜底，此时相当于单分支，必须考虑是否有逻辑漏洞

```go
if 布尔值 {
    // true时语句
} else if 布尔值 {
    // true时语句
} else if 布尔值 {
    // true时语句
}
```

- 嵌套：工程上一到两层即可，尽量不超过三层，嵌套过多不利于阅读

```go
if 布尔值 {
    // true时语句
} else {
    if 布尔值 {
        // true时语句
    } else {
        // false时语句
    }
}
```

- switch写法：Go中switch没有穿透效果。表达式写法的作用是限定变量/常量在当前switch中使用
    - 多分支
        - 解决范围问题

```go
switch 默认布尔值（可不写，默认为true） {
case 布尔值:
    // true时语句
    fallthrough // 实现穿透效果（无视下一case条件直接执行下一层true时语句）需手动写该语句，仅穿透一层。
case 布尔值：
    // true时语句
default:
    // false时语句（兜底，可不写）
}
```

```go
switch 表达式（例如a:=100）; /*分号（;）后隐含默认布尔值true*/ {     // 初始化表达式后必须写分号;
case 布尔值：
    // true时语句
case 布尔值：
    // true时语句
default:
    // false时语句（兜底，可不写）
}
```

- 解决特定值问题

```go
switch 变量或常量 {
case 等于某值:  // 可写多值，使用逗号分隔
    // true时语句
case 等于某值:
    // true时语句
default:
    // false时语句（兜底，可不写）
}
```

```go
switch 表达式（例如a:=100）; a {     // 初始化表达式后必须写分号;
case 等于某值:  // 可写多值，使用逗号分隔
    // true时语句
case 等于某值:
    // true时语句
default:
    // false时语句（兜底，可不写）
}
```

- 循环：条件满足就反复执行，不满足就不执行或不在执行
