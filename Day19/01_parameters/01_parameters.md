# 形式参数
- 可以无形参，也可以多个形参
- 不支持形式参数的默认值
- 形参是局部变量

- 定义时，占位
  - 除了可变参数外，调用时提供相同且类型一致的实参

- 普通形参
  - 不支持缺省值
  - 传入实参时，只能按照位置必须对应好

- 可变形参variadic
  - 不支持缺省值
  - 剩余参数
  - 形参 (nums ...Type)，nums的类型是切片，元素类型Type
  - ...Type写法在混用时必须放在最后
  - Go不支持类似 js/python t1 (*[1,2]) == t1 (1, 2)的语法
  - 形参 (nums ...Type)，nums 的类型是切片，元素类型 Type
  - 没有类似 js/python t1 (*[1,2]) == t1 (1, 2)，Go 不支持
  - 形参列表中，如果有可变形参一定是唯一的，且在最后一个
  - 第一种 剩余参数
    - 按照位置剩余的收集起来放到切片中
    - t1 () [],0,0
    - t1 (100) [100],1,1
    - t1 (100, 200, 300, 400) [100 200 300 400], 4, 4
    - 提供的 1 个个实参收集起来放在 nums 切片中
  - 第二种 切片...
    - 切片不会分解装在nums里面

```txt
            s := [] int {3, 4}
            fmt.Printf ("% p, % p, % d, % d\n\n", &s, &s [0], len (s), cap (s))
```

    - t3 (1, 2, s...)
    - 如果你是切片...，共用底层数组
    - 本质是什么？相当于 header 的复制啊？
  - 第一种和第二种为可变参数传参时，不可混用

```go
        func Println (a ...any) (n int, err error) {
            return Fprintln (os.Stdout, a...)
        }
```

  - a 切片 [] any

```txt
        t := [] int {1, 3}
```

  - fmt.Println (t)
    - 第一种按照位置传参的方式，1 个实参，不是切片传递
    - 所以 Println 要自己构建一个切片和底层数组，数组元素是 t
    - a [] any {t}
  - fmt.Println (t...)
    - a 和 t 共用底层数组，a 和 t 的 header 内容一样
    - a [] any {1, 3}
  - fmt.Println

            for _, v := range a {
            v => os.stdout
