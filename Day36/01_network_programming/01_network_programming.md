# 网络编程 IO
- 物理设备（路由器、交换机、光纤、双绞线等）、协议（电信号等规则）
- 常见TCP/IP协议族
  - 网络协议
      - TCP
      - UDP
      - port 想象成信箱号码，信件到楼了可以通过端口投递到port，port要和应用程序捆绑
        - 发送时需要端口，客户端发送，发送时随机挑选一个未被使用的大整数端口号
        - 接收时也需要端口，接收端一定要指定绑定端口（固定）
  - IP协议
      - IP 端到端的问题，从网络设备的一端到另一端，源IP地址、目标IP地址，路由route
        - 主机到主机的路由，投递数据包。想象成 楼到楼，没有放到应用程序的信箱里
        - 路由有很多条路
          - 最近
          - 最优
        - 127.0.0.1 本机回环IPV4地址
  - socket套接字的另一个意思，地址:端口 对

- TCP编程
  - 早期编程就要利用协议提供的编程接口API（提供的编程库函数）
  - Unix一个分支BSD，Barkley socket，高级网络编程接口，在协议编程接口之上，套用简单的套路程序就完成了网络编程
  - socket编程
      - 为什么叫socket？插座
      - 网络通信要解决的是什么？一个应用程序和另一个应用程序通信
      - 每个应用程序都需要自己的socket对象

      - 选择协议
        - 协议族
          - 名称             协议
          - AF_INET         IPV4，缺省值
          - AF_INET6        IPV6
          - AF_UNIX         Unix Domain Socket，Windows没有
        - Socket类型
          - 名称             协议
          - SOCK_STREAM     面向连接的流套接字。缺省值，TCP协议（目前仍是开发首选）
          - SOCK_DGRAM      无连接的数据报文套接字。UDP协议
          - TCP协议是流协议，也就是一大段数据看作字节流，一段段持续发送这些字节
            - 有先后顺序，保证包序不能乱，丢包时会要求包重发，控制不能多发，不能重发，不能丢包，不能乱序
            - 字节流可以看作字节切片
          - UDP协议是数据报协议，每一份数据封在一个单独的数据报中，一份一份发送数据
            - 报文之间没有关系
            - 不需要回复是否收到
  - C/S模型
      - Client 客户端
        - 提出请求（发出请求），主动向对端发出请求，等着对端返回数据
      - Server 服务端
        - 响应请求（提供服务），被动接收请求，将数据组织好，返回给客户端
      - 客户端程序用什么语言写，程序在什么操作系统上运行 都没关系，遵循协议就行
