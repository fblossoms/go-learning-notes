    - 可以有多个口（并行）
      - 并行，同时处理，并行是并发的一种解决方案，并行会带来成本提高
      - 也可以队列、争抢

# 并发编程
- 假并行和真并行
  - 假并行
    - 单个CPU利用时间片执行多个进程
  - 真并行
    - 多核CPU执行多个进程
- 进程
  - 操作系统OS，一种重要的功能：进程管理
  - 运行中的程序，运行时在内存中为这个程序分配包括内存空间的运行实例
  - 作业是运行的任务，这些任务就称为进程
  - 源代码写好编译好生成可执行文件.exe二进制符合WinPE标准的二进制格式，躺在磁盘上，称为程序
  - 程序可以被OS加载，为它开辟内存空间分配数据结构，分配pid，内存中的实例称为进程
  - 初始化操作做完，OS要找到当前进程的固定内存位置，即入口地址，最终会指向到main函数
  - 每个进程霸占内存，OS要哄进程开心，会让进程认为进程自己独占着内存，32位为例 虚拟地址0 ~ 2^32，但是给进程分页分配的。
    - f(虚拟地址) -> 真实的物理地址
    - 内存不够用时会报缺页异常，OS才会分配页，然后加载这部分数据
    - 物理内存告急时，会使用虚拟内存swap，拿硬盘空间做内存空间用
    - 一旦要的数据在虚拟内存中，就要访问这个慢的要死的假内存的数据加载到内存中，置换不常用的数据到swap
  - OS操作系统管理，按照进程管理，进程的和进程之间不相干，相互独立。要暴露出端口才能相互访问，如网络端口

- 线程
  - 只有进程自身，遇到IO时，只能等，即阻塞（进程卡住了，不能动了），此时不能指向下一条指令
    - 就这个进程来说，如果还有很多不同的任务要处理的话，那不能处理
  - 为了增加进程并行的处理能力，引入线程。以前指令是进程调度的，现在OS按照线程调度。颗粒度降低
    - CPU上跑的是线程，线程对应的指令，进程进程至少会提供一个线程，称为主线程
  - 有了线程之后，进程就成为管理单位（内存资源、IO资源等），线程成为运行的最小单位

  - 同一个进程内，线程之间的关系相当于 省与省的关系，每个线程都在进程的内存中分配独立的内存资源，主要就是线程的栈，用于函数调用
    - 进程1的线程1和进程2的线程1有什么关系？   没有任何关系
    - 能不能用进程资源？                    能
    - 进程1能否使用进程2的资源               一般不能

# 线程状态机
- 线程的状态
  - 状态              含义
  - 就绪Ready         线程能够运行，但在等待被调度。线程可能刚刚创建启动，或刚刚从阻塞中恢复，或被其他线程霸占
  - 运行Running       线程正在运行
  - 阻塞Blocked       线程等待外部事件发生而无法运行，如I/O操作
  - 终止Terminated    线程完成，或退出，或被取消

# 线程编程
- Python 源代码 先编译成字节码，在虚拟机运行字节码，实时翻译成当前CPU可以运行的指令
- python.exe xxx.py 启动一个解释器进程（虚拟机进程），主线程里面运行代码


  - __init__(self, group=None, target=None, name=None,
          - args=(), kwargs=None, *, daemon=None):
- 构造器，初始化方法
  - self露馅了，self这里表示当前实例自身，就当看不见
  - group 不用，保留
  - Thread(target=fn) 跳过第一个参数，按照名称给比较方便，就可以不按顺序
  - target 目标函数，干活的那个家伙，一般写函数名
  - name 线程名字就是你给的
  - args 目标函数的参数，按位置给
  - kwargs map 参数用键值对给
  - daemon daemon线程，主线程没事干了，环顾四周如果没有一个non-daemon线程，主线程退出，程序退出
    - Python默认主线程被设置为non-daemon
    - 如果创建线程时，daemon为None，子线程从父线程集成daemon属性

- Python 没有提供线程退出的方法，线程在下面情况时退出
  - 线程函数内语句执行完毕
  - 线程函数中抛出未处理的异常
