# Go协程创建和管理
- 语法
  - 普通函数调用：func(传参)
  - 创建协程：go func(传参) // 就会封装一个协程任务
- main函数也在一个协程中运行，main Goroutine主协程，主协程运行完后，进程结束（不会等其他的协程，没运行完的也都杀掉）
  - 生产中不可能人为计算其他进程需要跑多少秒来控制协程，所以需要引入 等待组 来管理
- 父子协程
  - 父协程
    - 创建其他协程的协程
  - 子协程
    - 被其他协程创建的协程

- loopclosure问题（1.22以前版本会触发）
  - 1、构建时如果用到循环变量，最好直接用 传参 拿走（值拷贝），该方法其他语言通用
  - 2、使用1.22版本及以后，i范围缩小，i局限于当前循环
