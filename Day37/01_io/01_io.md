# I/O（输入/输出）
- 从外部设备输入数据到内存 和 从内存输出到外部设备

- 外部设备都称为IO设备
  - 键盘、鼠标   纯输入
  - 打印机      纯输出
  - 显卡        纯输出（高速）
  - 网卡、磁盘   输入/输出

- 应用程序加载后变成进程，进程占用一定内存空间，进行划分
  - 一段 操作系统OS用，应用程序不能用，小一些，内核空间（用户无法访问，需要 系统调用特权指令），内核态
  - 一段 应用程序自己用，大一些，变成使用的堆栈都在这里，用户空间（用户指令），用户态
  - ring环
    - ring0   特权级别    可以访问所有内存空间
    - ring3   用户指令    只能访问用户内存空间
- IO两个阶段（以读取操作f.read()为例，底层调用syscall 系统调用函数）
  - 1、数据准备阶段。从设备读取数据到内核空间的缓冲区（通过 DMA直接内存访问，协处理器），什么时候完成不知道，完成了就用中断
  - 2、内核空间复制回用户空间进程缓冲区阶段（通过 DMA直接内存访问，协处理器）

# 同步、异步
- 看最终结果
  - 同步
    - 当下直接拿到最终结果
  - 异步
    - 当下未拿到最终结果，拿到一个凭据，凭借此凭据可以在某个时刻获得最终结果。至于到了这个交付时刻，收不收随你

# 阻塞、非阻塞
- 看行为卡不卡，不达目的就卡，阻塞了
  - 阻塞
    - 卡住了，不向后继续行动
  - 非阻塞
    - 卡不住，继续做其他事情

# 同步、异步和阻塞没关系，反之亦然，他们是不同的概念

# IO模型
- 同步
  - 同步阻塞：需要多线程配合使用
  - 同步非阻塞
  - IO多路复用（事件驱动IO）
    - OS提供功能
    - 多路选择器selector
      - 多路IO管理，监视多路，好了就通知拿走（读写还是要靠读写函数完成）
      - 选择什么？被管理的IO的某些路好了
      - 技术性能
        - 低性能
          - Linux、Unix、Windows
            - 提供select、poll
        - 高性能
          - Linux
            - Epoll
          - Windows
            - IOCP
          - BSD/Mac
            - kqueue
  - selector 定义的多路选择器
  - selector 注册了10路IO，都监视 读事件，分别是i1、i2、...、i10
  - select() 开始监视10路，里面多路选择器selector监视注册的10路，默认阻塞
    - 某一时刻，有5路第一阶段都完成了，select()返回

```txt
            for i in 5路的容器:
                data = i.recv()
```

```txt

```

  - fileobj
    - FileDescriptorLike：有fd文件描述符的对象，或fd整数文件描述符
  - events 你对 fileobj 的关注的事件们，r、w
  - data ？
  - 返回值 SelectorKey 类型对象
- selectors.select()阻塞到有一路或多路ok
  - 返回值 list[tuple[SelectorKey, _EventMask]] # [(元素0, 元素1)]，有多少元组就有多少路，当前这里有1路
