# 结构体序列化
- 把内存中的数据（立体）转换成 线性的序列，核心是数据，不关注方法
- 自定义类型属性差别很大，但是属性和值都是键值对结构，JSON序列化会把他们转化成JavaScript的对象类型
- 结构体一般在包里定义
  - 如果包外使用，一定要首字母大写
  - 如果要序列化，也一定要首字母大写
- 序列化标签
  - 使用序列化标签可以节省字节、带宽，提升服务器效率
  - 标签需要保证能看懂
  - 格式
    - 属性名 类型 `库要求的名称:"p1,p2"` // 单用p2时，逗号不可省略，否则就视为p1
      - p1 可选，表示字段名称（Json库），控制序列化（属性名 -> p1、反序列化（p1 -> 属性名）。
        - 若为减号 -，则不参与序列化、反序列化
      - p2 可选，omitempty，忽略空值，反序列化时，若是零值就不输出
        - false 0 "" nil
  - 标签配合Json序列化和反序列化 msgpack二进制序列化反序列化，ORM定义字段

- Json序列化的Go实现效率较低，可以使用社区第三方库实现，如easyjson、jsoniter、sonic等
