# ORM 必会
- 对象关系映射
  - Object面向对象  对象
    - 具有一类共同属性特征的事物抽象得到结构体（type类型）
    - 实例化出 一个个 实例，这些实例 都拥有 这些属性，可以实例化n个实例
  - 关系  关系数据表
    - 表是一种对若干行（实体）的抽象
      - 问题：需要定义几个抽象？
        - 1个
    - 若干字段，用来存放实体特征数据
    - 每一行 就是实实在在的实体
      - 问题：实体能有多少个？
        - n个
  - Mapping  映射
    - 结构体 -> 表关系
    - 属性 -> 字段
    - 实例 -> 实体
      - C增
        - 插入实例，保存到数据库表中对应一行
      - R查
        - 查询，返回一行或多行实体，转换成一个实例或多个实例
      - U改
        - 更新，实例保存instance.Save()，转换成update的SQL语句，到数据库中找到对应更新
      - D删
        - 删除，实例删除instance.Delete()，转换成delete的SQL语句，找到表中对应行进行删除

- 来源
  - 据说，有程序员不会写SQL，用面向对象的方式封装类，提供若干方法，调用类的实例的方法操作 => SQL语句 => 返回的结果也是对象，就可以用对象的方法操作返回的数据集了
- ORM效率略低于直接操作SQL语句，但是更适合面向对象开发，可以大胆使用。但是遇到耗时查询可以自定义
