# GORM
- 实现ORM功能
- 约定
  - 表名
    - 默认结构体小写+s，否则要实现一个接口Tabler的TableName() string方法
  - 字段约定
    - 属性首字母大写，否则没办法用
    - 默认约定规则，属性名使用大驼峰，例如 FirstName -> first_name
    - 如果不按照约定，例如 Fn要对应字段名first_name，要手动指定，利用标签tag
- 迁移
  - 从面向对象的类（属性定义） 生成 数据库中的表（字段和属性）
    - 结构体 -> 表关系
    - 属性 -> 字段
  - 问题：项目中表怎么来？
    - 1、手动创建
    - 2、SQL脚本（部署）
      - 构建模型，模型生成针对某数据库的表的SQL语句
      - 创建表，导出为SQL脚本
      - 脚本中还可以把初始数据insert进去
    - 3、面向对象构建类型（结构体）迁移
- 时间字段
  - time.Time、*time.Time返回的是string，对应出问题，DSN增加参数parseTime=true
  - 当前时区时间存入的时候ORM帮我们处理为UTC
    - 保证大家对数据库表中时间理解时区是一致的，多数采用UTC
    - 保证存入的时区和读取理解一致

  - 时间戳 int UTC的时间戳 => UTC时间 => Local() （推荐）
    - 到秒 t
    - Js 毫秒 T * 1000
      - 存入时，数据库字段中的时间就是当前时间的时间值
      - 读取时，数据库字段中的时间就被解读为当前时间
