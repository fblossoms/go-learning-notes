# 数组

- 特点
  - 长度不可变：定义数组时必须明确元素的个数（即长度必须是常量，不能是变量，不明确长度时是另一种类型：切片slice）
  - 内容可变：数组不能被定义为常量
  - 可索引：支持正索引，不支持负索引
  - 值类型：若元素未赋值，则用该元素类型的零值。
    - 若a := b // b是数组，a就是拷贝了一份b的值
    - 数组是值类型，赋值，形参传实参、形参，拿回返回值都会发生值拷贝
    - 缺点：产生过多临时副本（内存），频繁引起垃圾回收。可以改为指针传递，用同一个指针指向的内容，改地址的内容
  - 顺序表：数组由顺序表实现，数组地址就是索引0元素的地址

- 定义时可以只定义类型，不进行初始化，不初始化默认使用对应元素类型的零值

- 一些概念
  - 长度len：元素的个数
  - 容量cap：连续空间，开辟的内存空间内可以容纳多少个元素
  - 对Go数组，长度等于容量，因为初始化时就确定了。用长度来申请空间，元素有多少个就申请容纳多少元素的空间。对于多维，若没指定行索引，则反映
    第一维的长度（行数）

- 字符串数组
  - 一维字符串数组仍是顺序表，间隔16字节（string header标头值 / 元数据，分为长度len + 底层数组的指针各8字节）
  - 字符串不存储在数组内，而是通过存储在长度len + 底层数组指针访问

# 值拷贝（整数类型、数组类型、指针类型（大整数））

- 值本身完整的复制了一次
- 发生情况
  - 赋值
  - 形参传实参
  - 取返回值
- 缺点：产生过多临时副本（内存），频繁引起垃圾回收。可以改为指针传递，用同一个指针指向的内容，改地址的内容
