# 切片 slice（Go中重要的数据结构类型，和map占到了开发的80%以上）

- 表现为顺序表
- 表现为长度可变、内容也可变
- 表现为引用类型

- 本质（由两部分组成）
  - 底层基于Go数组实现
  - SliceHeader 标头值、descriptor 描述符，结构体类型（元数据，大小为24字节）
    - （底层基于数组，但是想表达和数组不一样的对容量的利用）
    - ptr 指针类型，地址：本质是一个大整数，找数组首地址 -> 底层数组首地址（初始化时），Go没有提供直接的访问方法
      - 若为子切片写法，有可能ptr不是底层数组首地址
    - len int 长度：当前容器中元素的个数
    - cap int 容量：当前容器占用的格子数，最大容纳的元素数
      - 从ptr开始到底层数组结束还能容纳几个元素
    - （len、cap表达的是slice的当下情况，注意不一定表达的是底层数组的长度、容量）

- 定义方法
  - var a []int // 未初始化，有标头值，但ptr没指向（ptr==nil）
    - 切片可以用nil
  - 字面量（例如[]int{} []int{1, 2, 3}  []int{2:200}）
  - make（定义时无法给值，默认用元素类型对应的零值）
    - 参数1写类型（如[]int，[]string等）
    - 参数2写长度len，若没有参数3，cap也可以当len（cap >= len）
    - 参数3写容量cap，容量大于len时，表示切片可以增加元素数量直至cap == len

- 通过print(切片) 可印证打印了ptr。print一般不要使用
